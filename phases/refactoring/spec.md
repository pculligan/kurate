# Refactoring Phase Spec

## Goal

Use scripts and AI workflows to turn triage results into cleaner, more useful knowledge artifacts.

## Inputs

- triage manifests
- extracted Markdown
- metadata and analysis outputs
- grouped duplicate or overlap sets

## Outputs

- proposed canonical summaries
- merged or consolidated Markdown drafts
- improved information architecture proposals
- archive notes explaining why older material is being relegated

## Likely Near-Term Utilities

- prompt packs for summarization, overlap comparison, and canonical-page drafting
- helpers that assemble grouped input context for AI-assisted cleanup

## Success Criteria

- outputs are easier to read and maintain than the source material
- recommendations are traceable to the source corpus
- the phase helps produce a cleaner working set without destroying historical context

## Handoff

The output of refactoring feeds the warehousing phase and any curated downstream repository.
